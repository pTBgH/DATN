SYMPTOM:
  Backend middleware cannot access KEYCLOAK_BASE_URL and KEYCLOAK_REALM env vars

EVIDENCE:
  ✓ Pod has vars:     kubectl exec ... -- env | grep KEYCLOAK
  ✗ PHP cannot read:  env(), config(), getenv(), $_SERVER, $_ENV all return NULL
  ✗ Result:          Middleware builds malformed URL "/realms//protocol/..."

ROOT CAUSE:
  PHP-FPM isolation - environment variables are at Kubernetes pod level but not 
  propagated to PHP-FPM worker processes. This is a common issue with PHP containerization.

WORKAROUND APPLIED:
  Hardcoded URL in middleware for testing:
  $baseUrl = 'http://keycloak.security.svc.cluster.local:8080';
  $realm = 'job7189';

PERMANENT SOLUTIONS TO EXPLORE:

  2. Create init container that writes env vars to .env file
  3. Use Vault agent to inject secrets instead of env vars
  4. Modify Dockerfile to pass env vars to PHP at startup
  5. Use supervisor to export env vars before starting PHP-FPM
